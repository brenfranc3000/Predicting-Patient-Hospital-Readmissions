Predicting 30‑Day Hospital Readmissions

An ML-powered prototype using MIMIC-IV, batch DataOps, MLflow experiment tracking, and versioned models to support discharge risk assessment.

Brenda Franco • CheikhAbdoul • Taiwo Mudah • Sidharth Gadgil • Rohan Barai

Problem & Purpose

Why this system matters

Domain problem

  • Hospitals need better ways to identify patients likely to be readmitted within 30 days after discharge.
  • High readmission rates increase operational strain, worsen patient outcomes, and can trigger reimbursement penalties.
  • Risk depends on complex interactions across diagnoses, comorbidities, labs, medications, length of stay, and prior admissions.

ML system purpose

  • Predict 30-day readmission risk using MIMIC-IV patient, admission, diagnosis, and treatment data.
  • Use XGBoost as the primary model and logistic regression as the baseline for comparison.
  • Support discharge planning by giving clinicians an early risk signal they can act on.

Data Pipeline

What our pipeline implements

📥

Ingestion

Batch extraction from MIMIC-IV tables using Python, aligned with the proposal’s batch DataOps strategy.

✅

Validation

Schema checks, label completeness, valid ages, and sensible admission/discharge logic to protect data quality.

🧪

Features

Age, length of stay, prior admissions, diagnosis groupings, medication counts, and simple comorbidity signals.

🗂️

Versioning

Processed datasets are tagged and linked to runs so results stay reproducible across experiments.

Stored output

The pipeline produces a processed training table with one row per index admission, a binary 30-day readmission label, and versioned artifacts ready for model training and comparison.

ModelOps

Experiment tracking and model versioning

Tracked in each run

  • Run metadata: name, description, timestamp, model type, dataset version.
  • Preprocessing version and feature set used for training.
  • Hyperparameters for logistic regression and XGBoost.
  • Metrics: recall, precision, F1, ROC-AUC, plus confusion matrix artifact.

Versioning strategy

  • Logistic regression serves as a transparent baseline for comparison.
  • XGBoost is the primary model because it better handles nonlinear interactions and missingness in EHR-style data.
  • Best runs are promoted into a registry with version labels such as readmit_xgb_v1.
  • Artifacts and descriptions support collaboration, traceability, and future deployment.
